# 词库备份功能设计

## 概述

为 IELTS Vocab 应用增加词库自动备份功能，支持按周期（每天/每周/每月）自动将学习进度导出为本地 JSON 文件，并提供从备份文件恢复数据的能力。

## 需求摘要

- **备份方式**：本地文件导出（JSON）
- **备份模式**：仅自动周期备份（每天/每周/每月可选）
- **备份数据范围**：仅学习进度（Hive `learning_records` Box）
- **备份通知**：备份成功后发送本地通知提醒用户
- **恢复功能**：支持从备份 JSON 文件恢复学习进度
- **UI 布局**：内嵌设置卡片，直接在个人中心页展示配置区域

## UI 设计

### 整体布局

在 ProfileScreen 现有「数据备份」设置项位置，将原来的 `ListTile`（点击弹 SnackBar 占位）替换为展开式内嵌配置区域。

### 配置区域结构

配置区域包含以下元素，从上到下排列：

1. **自动备份开关** — `Switch` 控件，默认关闭。标题"自动备份"，右侧开关
2. **周期选择** — 三个 `ChoiceChip`（每天 / 每周 / 每月），仅在开关打开时可交互，默认选中"每周"
3. **上次备份时间** — 小字辅助文本，格式"上次备份: 2026-05-05 09:00"，未备份时显示"未备份"
4. **恢复数据入口** — 文字按钮"恢复数据"，点击后打开系统文件选择器

配置区域使用 `Card` 包裹，与个人中心其他设置卡片风格一致。

### 恢复确认弹窗

选择备份文件后，弹出确认对话框：
- 标题："恢复学习进度"
- 内容："将从备份文件恢复 N 条学习记录，已有记录将被更新，本地其他记录不受影响。确定继续？"
- 按钮："取消" / "确定恢复"

### 交互细节

- 开关关闭时，周期选择 Chip 置灰不可点击
- 开关打开后，立即启用周期选择
- 恢复操作完成后，`SnackBar` 提示"已恢复 N 条学习记录"或错误信息

## 数据设计

### 备份文件格式

文件名：`ielts_backup_YYYYMMDD_HHmmss.json`

```json
{
  "version": "1.0",
  "exportDate": "2026-05-06T09:00:00Z",
  "recordCount": 42,
  "records": [
    {
      "wordId": "word_001",
      "ease": 2.5,
      "interval": 7,
      "repetitions": 3,
      "nextReview": "2026-05-13T00:00:00Z",
      "lastReview": "2026-05-06T09:00:00Z"
    }
  ]
}
```

### 存储位置

备份文件存放在应用外部存储目录（`getApplicationDocumentsDirectory()`），确保用户可通过手机文件管理器访问。

### 设置持久化

在现有 Hive `settings` Box 中新增以下字段：

| Key | 类型 | 默认值 | 说明 |
|-----|------|--------|------|
| `backup_enabled` | `bool` | `false` | 自动备份开关 |
| `backup_period` | `String` | `"weekly"` | 备份周期：`daily` / `weekly` / `monthly` |
| `backup_last_time` | `String` | `null` | 上次备份时间 ISO 8601 字符串 |

### 备份历史管理

保留最近 5 个备份文件。每次执行备份后，扫描备份目录中所有 `ielts_backup_*.json` 文件，按时间排序，删除超出 5 个的最旧文件。

## 自动备份调度

### 调度机制

不依赖后台定时任务插件，采用**启动时检查**策略：

1. App 每次启动时，读取 `backup_enabled` 和 `backup_last_time`
2. 根据当前周期配置判断是否到期：
   - `daily`：上次备份距今 > 24 小时
   - `weekly`：上次备份距今 > 7 天
   - `monthly`：上次备份距今 > 30 天
3. 到期则执行备份，更新 `backup_last_time`

### 执行时机

- 在 App 启动后延迟 10 秒执行（避免影响启动性能）
- 使用 `Future.delayed` 实现

### 通知

- 备份成功后，通过 `flutter_local_notifications` 发送一条本地通知
- 通知内容："词库备份完成"
- 通知标题："IELTS Vocab"
- 备份失败时不发通知，下次启动自动重试

## 恢复功能

### 恢复流程

1. 用户在配置区域点击"恢复数据"
2. 调用系统文件选择器（`file_picker` 插件），过滤 `.json` 文件
3. 用户选择文件后，解析 JSON 内容
4. 弹出确认对话框，显示将恢复的记录数量
5. 用户确认后，逐条写入 `learning_records` Box，采用 merge 策略：仅覆盖备份文件中存在的记录，本地独有的记录保留不动
6. 完成后 `SnackBar` 提示"已恢复 N 条学习记录"

### 错误处理

- 文件格式不合法：提示"备份文件格式错误"
- 文件版本不匹配：提示"备份文件版本不兼容"
- 读取/写入失败：提示"恢复失败，请重试"

## 非功能需求

- 备份文件大小预估：1000 条学习记录约 50KB
- 备份操作不影响 UI 响应（异步执行）
- 恢复操作需显示进度（如果记录数 > 100）
